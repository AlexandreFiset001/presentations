Open Week 2018

Introduction et présentation

Objectifs de la semaine

  • Faire un pas dans l'univers de l'Open Source
  • S'impliquer dans un projet de long terme
  • Apprendre de nouveaux outils et technologies
  • Avoir un exemple d'application de vos études
  • Passer un bon moment

Déroulement de la semaine

  • Petit déjeuner le matin
  • Contribution au projets au matin et l'après midi
  • Présentations et/ou activité chaque jour
  • Repas chaud à midi (RestoU typiquement)
  • Activités de détentes quand vous le désirez

Attention, les salles informatiques ont été refaites il y a peu, donc ne mangez/buvez pas en salle s'il vous plaît

Les projets de cette année

  • Syllabus interactif : backend et contenu
  • Mini station météo sur raspberry pi
  • Créer des exercices blockly sur INGInious
  • Un jeu mobile pour le Kap Hot
  • Plugin ICTV/INGInious

Syllabus interactif

  • Nouvelle fonctionalité (ex: double panel pour l'édition, système de traduction, etc.)
  • Exercice python avec un context très développé

Mini station mété sur raspberry pi

  • Récupération de différents type de donnée via capteur
  • Traitement des données pour créer des graphiques
  • Affichage des résultats sur un serveur web

Exercices blockly

  • Blockly est un language de programmation basé sur des bloc
  • Utile pour l'apprentissage de l'informatique chez les jeunes

Plugin ICTV/INGInious

  • ICTV est le programme qui affiche les slides sur les écrans du département (entre autre)
  • Le plugin peut être uniquement sur ICTV
  • ou alors être lié avec d'autre programme comme INGInious
  • Exemple : afficher des stats rigolote d'INGInious (tâche la plus raté, la plus plus essayée, etc.)

Organisation pratique

  • Pour facilité l'organisation, vous serez séparer en groupe dans chaque projet
  • Au milieu de la semaine, vous aurez l'opportunité de changer de projet

Nous avons fait une répartition initiale en fonction de vos réponse au formulaire. Cette répartition n'est pas fixe, si vous voulez changé, venez me voir après la présentation.

Avant de commencer à travailler

En fonction de la répartition, vous allez assister à une présentation plus détaillée du projet dans lequel vous êtes. Après ces présentations, vous aurez une présentation Python pour ceux qui ne connaissent pas et qui en ont besoin
